1. Написать реализацию сумматора, интерфейс которого представлен в файле adder.v
Написать тестбенч для проверки правильности работы сумматора. Перебрать все возможные 
комбинации чисел на каждом из входов.
2. Написать реализацию алу, интерфейс которого представлен в файле alu.v
Написать тестбенч для проверки правильности работы алу. Перебрать все возможные 
комбинации чисел на каждом из входов для каждой операции. Операции которые должен выполнять алу и 
соответствующие им значения i_control прописаны в файле alu.v в виде параметров. 
o_zf - флаг нуля, устанавливается в единицу, когда все биты o_result равны нулю.
3. Написать реализацию модуля управления алу, интерфейс которого представлен в файле 
aluControl.v Принцип работы даного модуля описан в "Computer organization and design"
на стр. 301
4. Написать реализацию модуля озу на флип-флопах, интерфейс которого представлен в файле
ram.v
5. Написать реализацию модуля пзу, интерфейс которого представлен в файле
rom.v Эта память должна быть асинхронной. Считывание происходит в том же цикле тактовой чатоты, 
что и подача адреса считывания. Должна быть возможность записи ячеек памяти в начале симуляции
содержимым файла rom_init.dat Написать тестбенч для проверки инициализации rom содержимым
rom_init.dat
